Presigned URLs provide secure, time-limited access to files stored in S3-compatible storage without requiring recipients to have AWS credentials or authentication. Closeness Decoder uses presigned URLs to enable secure file sharing through friend URLs.

## Overview

Presigned URLs are temporary URLs that grant access to private S3 objects. They include authentication information in the URL parameters, allowing anyone with the URL to access the file until the URL expires.

<Note>
  Presigned URLs are cryptographically signed using your S3 credentials, ensuring that only URLs generated by your application can access your files.
</Note>

## How Presigned URLs Work

<Steps>
  <Step title="File upload to S3">
    File is uploaded to private S3 bucket with restricted access
  </Step>
  <Step title="Presigned URL generation">
    Backend generates a presigned URL with configurable expiration time
  </Step>
  <Step title="URL distribution">
    Presigned URL is shared with recipients via friend code
  </Step>
  <Step title="Direct S3 access">
    Recipients access file directly from S3 using the presigned URL
  </Step>
  <Step title="Automatic expiration">
    URL becomes invalid after expiration time, preventing further access
  </Step>
</Steps>

## Implementation

Closeness Decoder uses the AWS S3 SDK's presigner to generate time-limited URLs:

```java S3ServiceImpl.java
@Override
public ResponseEntity<?> getPreSignedUrl(String key) {
    GetObjectRequest request =
            GetObjectRequest.builder()
                    .bucket(s3Properties.getBucketName())
                    .key(key)
                    .build();

    GetObjectPresignRequest presignRequest =
            GetObjectPresignRequest.builder()
                    .signatureDuration(Duration.ofMinutes(45))
                    .getObjectRequest(request)
                    .build();

    String preSignedUrl =
            s3Presigner.presignGetObject(presignRequest).url().toString();
    return new ResponseEntity<>(preSignedUrl, HttpStatus.OK);
}
```

<Info>
  The presigned URLs are configured with a 45-minute expiration time via `Duration.ofMinutes(45)`. This provides a balance between security and usability.
</Info>

## S3 Presigner Configuration

The `S3Presigner` bean is configured alongside the S3 client:

```java S3Config.java
@Bean
public S3Presigner s3Presigner(S3Properties s3Properties) {
    return S3Presigner.builder()
            .region(Region.of(s3Properties.getRegion()))
            .credentialsProvider(StaticCredentialsProvider.create(
                    AwsBasicCredentials.create(
                            s3Properties.getAccessKey(),
                            s3Properties.getSecretKey()
                    )
            ))
            .endpointOverride(URI.create(s3Properties.getEndpoint()))
            .build();
}
```

## Presigned URL Structure

A typical presigned URL includes:

```
https://s3.us-west-000.backblazeb2.com/bucket-name/2026-03-03/uuid.pdf?
X-Amz-Algorithm=AWS4-HMAC-SHA256&
X-Amz-Credential=KEY_ID%2F20260303%2Fus-west-000%2Fs3%2Faws4_request&
X-Amz-Date=20260303T120000Z&
X-Amz-Expires=2700&
X-Amz-SignedHeaders=host&
X-Amz-Signature=abc123...
```

<Accordion title="URL Parameter Breakdown">
  - **X-Amz-Algorithm**: Signing algorithm used (AWS4-HMAC-SHA256)
  - **X-Amz-Credential**: Access key and scope information
  - **X-Amz-Date**: Timestamp when URL was generated
  - **X-Amz-Expires**: Expiration time in seconds (2700 = 45 minutes)
  - **X-Amz-SignedHeaders**: Headers included in signature
  - **X-Amz-Signature**: Cryptographic signature verifying authenticity
</Accordion>

## Integration with Friend URLs

Presigned URLs are used within the friend URL system:

1. User uploads a file
2. File is stored in S3
3. Presigned URL is generated for the file
4. Friend URL record stores the presigned URL
5. Friend URL is returned to user for sharing
6. Recipients access the file via friend code, which resolves to the presigned URL

```java S3ServiceImpl.java
// Inside sendObjectAndGeneratePresignedUrl method
uploadObject(file, key);
log.info("File uploaded to S3");
response = getPreSignedUrl(key);
// ... rest of implementation
String sourceUrl = response.getBody().toString();
FriendUrl friendUrl =
        FriendUrl.builder()
                .userId(userId)
                .sourceUrl(sourceUrl)  // Presigned URL stored here
                .sourceKey(key)
                .expiryTimeMinutes(60)
                .createdAt(now)
                .expiresAt(now.plusMinutes(60))
                .isActive(true)
                .build();
```

## Expiration Management

Closeness Decoder uses two layers of expiration:

### 1. Presigned URL Expiration

Presigned URLs expire after 45 minutes:

```java
.signatureDuration(Duration.ofMinutes(45))
```

After expiration, the URL returns a 403 Forbidden error from S3.

### 2. Friend URL Expiration

Friend URL database records track a 60-minute expiration:

```java
.expiryTimeMinutes(60)
.createdAt(now)
.expiresAt(now.plusMinutes(60))
.isActive(true)
```

<Warning>
  The presigned URL expires **before** the friend URL record (45 minutes vs 60 minutes). This means users accessing the friend URL near the end of its validity period may receive an expired presigned URL.
</Warning>

## Redis Caching

Presigned URLs are cached in Redis along with friend URL mappings:

```java RedisCacheService.java
public void createFriendUrl(String key, String value) {
    log.info("creating cache for friend code with source url");
    redisTemplate.opsForValue().set(key, value, Duration.ofMinutes(60));
}
```

This provides fast access to presigned URLs without database queries, but requires cache invalidation if URLs need to be regenerated.

## API Endpoints

Generate a presigned URL for a specific S3 object key. Requires authentication.

**Request:**
```
GET /api/presigned-url/2026-03-03/a1b2c3d4-e5f6-7890-abcd-ef1234567890.pdf
Authorization: Bearer jwt_token
```

**Response:**
```
https://s3.us-west-000.backblazeb2.com/bucket/2026-03-03/a1b2c3d4-e5f6-7890-abcd-ef1234567890.pdf?X-Amz-Algorithm=...
```

**Controller Implementation:**
```java S3Controller.java
@GetMapping(path = "/presigned-url/{key}")
public ResponseEntity<?> generatePresignedUrl(
        @PathVariable("key") String key
) {
    return s3Service.getPreSignedUrl(key);
}
```

## Error Handling

When presigned URLs expire or are invalid:

```java
// S3 returns 403 Forbidden for expired presigned URLs
<Error>
  <Code>AccessDenied</Code>
  <Message>Request has expired</Message>
  <Expires>2026-03-03T12:45:00Z</Expires>
  <ServerTime>2026-03-03T13:00:00Z</ServerTime>
</Error>
```

Your application should handle these errors gracefully:

```javascript
fetch(presignedUrl)
  .then(response => {
    if (response.status === 403) {
      // URL has expired
      alert('This link has expired. Please request a new one.');
    }
  });
```

## Best Practices

### Choose Appropriate Expiration Times

Balance security and usability:

- **Short-lived (5-15 minutes)**: For sensitive documents
- **Medium-lived (30-60 minutes)**: For general file sharing (current implementation)
- **Long-lived (several hours)**: For downloads that may take time

### Regenerate URLs When Needed

Implement URL regeneration for expired links:

```java
if (friendUrl.getExpiresAt().isBefore(LocalDateTime.now())) {
    // Generate new presigned URL
    ResponseEntity<?> newUrlResponse = getPreSignedUrl(friendUrl.getSourceKey());
    String newSourceUrl = newUrlResponse.getBody().toString();
    
    // Update friend URL record
    friendUrl.setSourceUrl(newSourceUrl);
    friendUrl.setExpiresAt(LocalDateTime.now().plusMinutes(60));
    friendUrlRepository.save(friendUrl);
}
```

### Monitor URL Usage

Track presigned URL access patterns using Kafka events:

```java
kafkaProducer.publishClickEvent(
        String.valueOf(friendCode), clickedAt);
```

This helps identify:
- Popular sharing times
- Expired link access attempts
- Potential security issues

### Invalidate Compromised URLs

If a presigned URL is compromised:

1. Mark the friend URL as inactive:
   ```java
   friendUrl.setIsActive(false);
   friendUrlRepository.save(friendUrl);
   ```

2. Delete the cached URL:
   ```java
   redisTemplate.delete(friendCode);
   ```

3. Optionally, delete the S3 object:
   ```java
   s3Service.deleteObject(friendUrl.getSourceKey());
   ```

## Limitations

<Warning>
  **URL Sharing**: Once generated, a presigned URL can be shared with anyone until it expires. There's no way to track or limit who accesses it.
</Warning>

<Warning>
  **Cannot Revoke**: Presigned URLs cannot be revoked before expiration. If a URL is compromised, you must wait for it to expire or delete the underlying S3 object.
</Warning>

<Note>
  **Clock Skew**: Ensure server clocks are synchronized with NTP. Clock skew can cause presigned URLs to be rejected by S3.
</Note>

## Advanced: Presigned URL for Uploads

While Closeness Decoder currently uses direct uploads, you can also generate presigned URLs for uploads:

```java
PutObjectRequest putObjectRequest = PutObjectRequest.builder()
        .bucket(bucketName)
        .key(key)
        .contentType("application/pdf")
        .build();

PutObjectPresignRequest presignRequest = PutObjectPresignRequest.builder()
        .signatureDuration(Duration.ofMinutes(10))
        .putObjectRequest(putObjectRequest)
        .build();

PresignedPutObjectRequest presignedRequest = 
        s3Presigner.presignPutObject(presignRequest);

String uploadUrl = presignedRequest.url().toString();
```

This allows clients to upload directly to S3, bypassing your server entirely.

## Comparison: Presigned URLs vs Direct Backend Proxy

| Aspect | Presigned URLs | Backend Proxy |
|--------|---------------|---------------|
| **Performance** | Direct S3 access, no server load | All traffic through server |
| **Bandwidth** | Free for your server | Counts against server bandwidth |
| **Security** | Time-limited, self-contained | Can add additional validation |
| **Tracking** | Limited to access time | Can log full request details |
| **Revocation** | Cannot revoke early | Can revoke anytime |
| **Complexity** | Simple implementation | Requires streaming logic |

<Info>
  Closeness Decoder uses presigned URLs for their performance benefits and to reduce server load.
</Info>
