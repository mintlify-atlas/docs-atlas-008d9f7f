## POST /api/upload

Uploads a PDF file to S3-compatible storage (Backblaze B2). The file is stored with a unique key based on the current date and a UUID.

### Authentication

Requires JWT authentication. Include the JWT token in the `Authorization` header:

```
Authorization: Bearer <token>
```

### Request

<ParamField body="file" type="file" required>
  The PDF file to upload. Must be a valid PDF file (Content-Type: application/pdf).
</ParamField>

<Warning>
  File size must not exceed 10MB. Only PDF files are accepted.
</Warning>

### Response

<ResponseField name="message" type="string">
  Success message indicating the file was saved successfully
</ResponseField>

### Example Request

```bash cURL
curl -X POST https://api.closeness-decoder.com/api/upload \
  -H "Authorization: Bearer <your_jwt_token>" \
  -F "file=@/path/to/document.pdf"
```

### Implementation Details

- Files are stored with keys in the format: `YYYY-MM-DD/<uuid>.pdf`
- The endpoint validates file presence and content type before processing
- Uses AWS SDK for S3-compatible storage operations
